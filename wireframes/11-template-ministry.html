Ministry Template

The landing page for a partner ministry — pastor identity, content, series, and a way for users to engage. Bound to URL pattern /ministry/[slug]. One template serves all twenty-plus ministry pages; the slug input parameter swaps the content per page.

One sentence

Ministry pages establish trust and identity for partner pastors — but use TrueVi branding only. Never adopt the ministry's colors, fonts, or visual brand. Ministry preference is a search and recommendation boost, not a wall.

Branding rule (from CLAUDE.md)

Ministry pages use the Abyss Teal palette exclusively. The ministry's logo appears in the Hero, but no other brand assets (colors, fonts, chrome) cross over. TrueVi is the platform; ministries are partners.

Annotated walkthrough

1
Breadcrumbs Standard nav aid — auto-generated from the route. Browse Ministries page is reachable one click up.
2
Hero-Ministry · TrueVi-branded Ministry logo + name + pastor + tagline + stats. Background gradient is TrueVi's Abyss Teal — not Mercy Hill's brand. The eyebrow "Partner Ministry" reinforces that this is a TrueVi page surfacing a partner, not the ministry's own site. Stats are auto-computed from content_items.
3
PastorCard · authored content Avatar, name, role, and bio come from the ministries + pastors tables. Editorial content, not Library-generated. Updated through the CMS / ministry-onboarding workflow.
4
Faith Stage chips · ministry alignment The seven Faith Journey stages (M1.1 §6) with this ministry's aligned stages highlighted. Pulled from ministries.faith_stage_alignment. Helps users self-identify whether this ministry fits where they are. Clicking a highlighted chip filters the rails below.
5
SeriesRail · ministry-scoped All series from this ministry. Newest first, "Complete" badge on finished arcs. Click navigates to /ministry/[slug]/series/[seriesSlug] (Series Template, M3.7).
6
Dynamic content sections · ministry-filtered All rails here filter to ministry == mercy-hill. Library decides which rails to surface based on the visiting user's signals. Returning user with high engagement on Mercy Hill gets personalized rails ("Because you've been exploring..."). Cold-start visitor gets discovery rails ("Most popular," "Latest").
7
Ministry-scoped Ask Anything Same AskAnythingBlock component, but the submission destination is /search?ministry=mercy-hill&q=... — keeping the user inside the ministry's content while still benefiting from the list-style search. Notably, this CTA does NOT trigger a Search Response (which is global); it routes to the scoped Search Template (M3.3).

Template definition

{
  "templateId": "ministry-v2",
  "pageType":   "ministry",
  "name":       "MinistryTemplate v2",
  "inputs": [
    { "name": "ministrySlug", "type": "slug", "required": true }
  ],
  "slots": [
    {
      "slotId": "breadcrumbs",
      "kind":   "FIXED",
      "component": "Breadcrumbs"
    },
    {
      "slotId": "hero",
      "kind":   "FIXED",
      "component": "Hero",
      "instance": {
        "config": { "variant": "Hero-Ministry" },
        "query":  { "contentType": "ministry", "mode": "auto", "filters": { "ministry": "${input.ministrySlug}" } }
      }
    },
    {
      "slotId": "pastor",
      "kind":   "FIXED",
      "component": "PastorCard",
      "instance": { "config": { "showBio": true, "showAffiliation": true } }
    },
    {
      "slotId": "faith-stages",
      "kind":   "FIXED",
      "component": "FilterChips",
      "instance": { "config": { "filterType": "faithStage", "multi": false } }
    },
    {
      "slotId": "series",
      "kind":   "FIXED",
      "component": "SeriesRail",
      "instance": {
        "config": { "layout": "horizontal-scroll", "showAllLink": true },
        "query":  { "mode": "auto", "sort": "recent", "filters": { "ministry": "${input.ministrySlug}" } }
      }
    },
    {
      "slotId": "content-sections",
      "kind":   "DYNAMIC",
      "allows": ["ContentRail"],
      "minCount": 2,
      "maxCount": 6,
      "slotInput": { "ministry": "${input.ministrySlug}" }   // pre-applies ministry filter to every fill
    },
    {
      "slotId": "ask-anything",
      "kind":   "FIXED",
      "component": "AskAnythingBlock",
      "instance": {
        "config": { "variant": "hero-card", "scopeTarget": "ministry", "submitDestination": "search" }
      }
    }
  ]
}

Template input · slot variables

One template, many pages. The ministry-v2 template takes a single typed input — ministrySlug — and serves all twenty-plus ministry pages. Slot configs use ${input.ministrySlug} substitution to bind the input into queries at render time.

InputTypeUsed in
ministrySlugslug · requiredHero query, SeriesRail query, content-sections slot input, AskAnything scope

When the URL is /ministry/mercy-hill, the renderer resolves ministrySlug = "mercy-hill" from the route, passes it as routeParams.ministrySlug in the compose-page request, and the Library applies it everywhere the template references ${input.ministrySlug}.

Ministry-preference boost · search semantics

Ministries are a boost, not a wall

Visiting a ministry page increases the user's visitedMinistries[mercy-hill].recencyScore. That score feeds into Library content scoring system-wide — Mercy Hill teachings rank higher in global recommendation contexts for this user going forward. But the best content from any ministry always wins when relevance dominates the score. Ministry preference is a 10% boost (see CLAUDE.md relevance scoring).

Library compose-page · ministry routing

POST /library/compose-page
{
  "pageType":    "ministry",
  "route":       "/ministry/mercy-hill",
  "routeParams": { "ministrySlug": "mercy-hill" },
  "identity":    { ... },
  "request":     { ... }
}

// Library
1. Resolve route → pageType=ministry, input ministrySlug=mercy-hill
2. Select template via TemplateSelector (likely ministry-v2 default)
3. For each FIXED slot: substitute ${input.ministrySlug} into queries
4. For DYNAMIC content-sections slot:
   a) Library decides 3 rails fit best for this user + ministry
   b) Each rail's query pre-filtered by ministry=mercy-hill via slotInput
   c) Library picks rail themes ("Most popular," "Personalized," "Latest")
5. Apply theological guardrails (always-on)
6. Page-level dedupe across rails
7. Return composition

// Side effect: hot-tier inference
   identities.visitedMinistries[mercy-hill].recencyScore ← bumped
   (informs future global recommendations)

Empty & edge states

Ministry not found

Renderer returns 404 with a fallback page: "This ministry isn't on TrueVi yet" + MinistryRail of similar ministries.

Ministry has < 5 published items

Dynamic content-sections may return fewer than the minimum count. Template's minCount=2 ensures we don't render a near-empty page; if even 2 rails can't be filled, an editorial fallback message renders: "More teachings from Mercy Hill coming soon."

Pastor bio not authored

PastorCard renders with avatar and name only; bio block is hidden. Editorial team is notified to backfill via the CMS.

Ministry permission status = denied

Content from this ministry is hidden platform-wide (theological guardrail or pulled permission). Visiting the URL returns 404.

Home Template

The default landing experience for truevi.tv/. Four slots shape the page; the Library fills them. The single most heavily-trafficked page on the platform — and the canonical example of the slot model in action.

One sentence

Home is a holding tank — one Hero, a flexible rail zone, an Ask Anything entry point, and an upgrade prompt for free users. The Library decides what fills the rail zone, in what order, and with what titles, per user, per session.

Annotated walkthrough

1
Hero slot · FIXED · Hero-Carousel Single FIXED slot at the top. Admin chose Hero-Carousel; Library decides which 5 items rotate through based on the user's inferredFaithStage, askAnythingHistory, and visitedMinistries. The "Featured for you" eyebrow is Library-authored — it's part of the response payload, not the template config.
2
Dynamic rails slot · DYNAMIC · 3-10 rails The heart of the page. Admin defined: min 3, max 10, allowed components are ContentRail · MinistryRail · SeriesRail. Library returned 5 rails for this user, including: a Continue Watching rail (signals: watchPatterns + content_state), a personalized rail framed around the user's recent subject exploration, a MinistryRail picking ministries with high faith-stage alignment, a trending rail (no personalization), and a short-format rail tuned to the user's preferredSessionMin. Each rail title is Library-authored.
3
Ask Anything slot · FIXED Always present, hero-card variant. Suggestion chips beneath are Library-generated based on askAnythingHistory and exploredSubjects. Empty for cold-start users — falls back to editorial default suggestions.
4
Upgrade teaser slot · FIXED · tier-gated Audience restricted to tier=new (free users). When Console's monetizationMode = "open-all", this slot renders empty in production but still shows in Console preview so admins can author it. When mode flips to tier-gated, this slot appears for free-tier users only.

Template variants

Multiple Home templates exist. The Library selects per user (M1.1 §8).

Template IDWhen selectedHow it differs from home-v3
home-v3 Default · all returning users Baseline. Carousel hero + 3-10 dynamic rails + Ask Anything + upgrade teaser.
home-discovery-v1 visitorType == first-visit Hero-Video with explainer instead of carousel. MinistryRail prioritized in dynamic slot. Larger Ask Anything block. No upgrade teaser.
home-power-v2 visitorType == power AND sessionCount > 30 Smaller hero (saves vertical space). More rails (up to 12). Compact card density.
home-plus-v1 tier == plus Premium framing in hero copy. No upgrade teaser. Adds a ScriptureBlock between rails. Larger Ask Anything.
home-v3-carousel Experiment variant for home-hero-carousel Same as v3 but the carousel auto-plays slower (10s vs 7s) and includes 7 slides instead of 5.

Cold-start behavior

First-visit users have zero behavioral signals. Library still produces a useful page:

  • Template selection: home-discovery-v1 is chosen via the first-visit selector rule (M2.2).
  • Hero: shows editorial-curated featured content + a framing tagline.
  • Dynamic rails: Library returns ministry-led exploration rails — "Discover ministries," "Most-watched this month," "Where to begin." No personalization yet; trending and editorial only.
  • Ask Anything: suggestion chips show "popular questions this week" instead of personalized.
  • Upgrade teaser: hidden — too early to push.

Search Template

The search results page. Used for both global and ministry-scoped search via URL parameter ?ministry=<slug>. Important distinction: this is the list-style search page. The free-form Ask Anything conversation lands on the Search Response Template (M3.4) instead.

One sentence

Search shows what TrueVi has matching the user's query — filterable, sortable, deep-linkable. Ministry scope is a banner + a Library content-pool filter, not a separate template.

Annotated walkthrough

1
Hero-Search · always present The search input lives in the hero. Suggested-query chips beneath are Library-generated from searchHistory and trending queries of users with similar profiles. On the ministry-scoped variant, the suggestions are restricted to ministry-relevant queries.
2
FilterChips · multi-select, URL-keyed Every chip toggle rewrites the URL: ?q=forgiveness&format=sermon&duration=under-15. The filter row's content is admin-authored — admins decide which filter types appear on this template.
2a
ContextBanner · ministry-scoped only Only renders when ?ministry=<slug> is present. Dismissing it removes the parameter and re-runs the search globally, preserving the ?q= value. This is how users escape a scoped search without losing their query.
3
Dynamic results slot The admin allows ContentGrid, MinistryRail, and SeriesRail here. The Library decides which components to use and in what order. For most queries it returns one large ContentGrid. For ambiguous queries it may return a MinistryRail or SeriesRail above the grid ("Ministries on this topic" / "Series on this topic"). The Library's choice depends on query intent and on how many matching ministries/series exist.
4
Ask Anything cross-sell · inline variant At the bottom of every search results page, the inline AskAnything block invites the user to convert their literal query into a guided question — driving them into the Search Response flow (M3.4). Eventbased: ask_anything_submitted with fromPageType: "search".

URL state model

ParameterTypePurpose
?q=stringThe query string. Persisted across filter changes and ministry-scope toggles.
?ministry=slugRestrict content pool to this ministry. Triggers ContextBanner render.
?format=enum/multiFilter by sermon / devotional / teaching / series
?duration=enumunder-15 / 15-30 / over-30
?recency=enumthis-week / this-month / this-year
?sort=enumrelevance (default) / recent / popular
?page=intPagination cursor for ContentGrid (if numbered)

Filter and sort changes preserve ?q= and ?ministry=. Dismissing the ContextBanner removes ?ministry= but preserves ?q= and active filters.

Library query · ministry-scoped vs global

// renderer → Library when ministry=mercy-hill
{
  "pageType":    "search",
  "route":       "/search",
  "routeParams": {
    "q":        "forgiveness",
    "ministry": "mercy-hill",
    "format":   ["sermon", "teaching"],
    "sort":     "relevance"
  },
  "identity": { ... },
  "request":  { ... }
}

The Library handles the ministry-scoping by adding a content-pool filter to all DYNAMIC slot fills. The renderer does not need to alter any of its hydration logic — the slot composition shape is identical to the global variant; only the contents differ.

Empty & edge states

No query (landing on /search with no ?q=)

Hero shows. Filters hidden. Results slot replaced with an editorial discovery panel: "Browse by subject" + popular subject chips + trending content rail.

No results

No teachings match "<query>"

Try different words, broaden your filters, or let us help you find what you need.

Click "Ask anything" below to describe what you're looking for in your own words.

Query too short (< 2 chars)

Renderer does not call Library. Results slot shows trending searches + the suggestion chips.

Ministry-scope with no matches

Results grid shows: "No teachings on this in this ministry. Search all ministries?" with a one-click toggle to global scope.

Personalization in Search

Search is heavily ranked by behavioral signal — the same query returns different orderings for different users.

  • Ministry preference · ministries the user has visited more recently rank higher
  • Format affinity · users with high sermon-completion get sermons surfaced first; devotional-leaners see devotionals
  • Faith stage alignment · content tagged for the user's inferredFaithStage gets a relevance boost
  • Duration preference · preferredSessionMin shapes the duration sort tiebreaker
  • Already-watched · completed content sinks lower; in-progress content rises

Confidence thresholds apply. Cold-start users see relevance-only ordering with no personalization boosts. As inferences sharpen, personalization weight increases.

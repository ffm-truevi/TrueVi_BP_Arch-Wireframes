Collections (Subject) Template

The page for a subject collection — forgiveness, anxiety, marriage, grief, doubt. Bound to /subject/[slug]. Subjects are the topical entry point that crosses all ministries and content types. Editorial framing + scripture anchor + filterable content grid + related subjects.

One sentence

Subject pages aggregate everything we have on a topic across ministries — anchored to scripture, framed editorially, and personalized for the visiting user's stage and life context.

Annotated walkthrough

1
Breadcrumbs Subjects sit at /subject/[slug]. Hierarchical subjects (parent → child) extend the breadcrumb chain.
2
Hero-Subject · subject identity Subject name in display type, subtitle tagline, content stats. Background gradient is the same Abyss Teal palette — no subject-specific theming. Stats auto-computed from content_items.subjects[].
3
Editorial framing · TextBlock Admin-authored prose that contextualizes the subject. This is not Library-generated. Lives on the subjects row as a locale-keyed field. Sets the spiritual tone for the page.
4
Scripture anchor The defining verse for this subject, pulled from subjects.scripture_anchors[0]. Editorial-authored. Distinct from Search Response scripture, which the Library picks per query.
5
FilterChips · sub-subject + format First few chips are sub-subjects (forgiving others, forgiving yourself, family wounds) — these refine the content pool. Later chips are content-format filters. The blend lets users either drill down into sub-topics or narrow by format, in one row.
6
Dynamic content slot · personalized rail above the grid Library returned two items: a personalized ContentRail labeled "For where you are" at the top (highest-relevance content for this user on this subject), then a ContentGrid showing all 124 teachings. The grid de-dupes against the personalized rail so the same content isn't shown twice.
7
Ministries on this subject MinistryRail filtered by subject. Helps users discover which partner ministries focus on this topic. Click navigates to /ministry/[slug] (M3.5).
8
Related subjects · Library-curated Subjects that frequently co-occur with this one across the user base. Computed from the content_items.subjects[] co-occurrence matrix. Updated cold-tier nightly. Functions like Netflix's "Because you watched..." but for topics, not titles.
9
Subject-scoped Ask Anything Same AskAnythingBlock component, but the prompt copy contextualizes the subject. Submission still creates a Search Response (M3.4) — the response can incorporate the subject scope as additional context.

The subjects table fuels this template

Unlike most templates, the Subject template draws heavily from editorial fields on the subjects row (M1.3 §15). The framing copy, scripture anchor, and sub-subjects are all editorial — admins author them through the CMS. The dynamic content slot is the only Library-personalized region.

Subject hierarchy

Subjects can have parent/child relationships via subjects.parent_subject_id. Examples: 

Forgiveness                              // top-level subject
├─ Forgiving others                      // child
├─ Forgiving yourself                    // child
├─ Forgiving when they're not sorry      // child
└─ Family forgiveness                    // child

Grief & Loss                            // top-level subject
├─ Loss of a parent                      // child
├─ Loss of a spouse                      // child
├─ Anticipatory grief                    // child
└─ Loss of faith                         // child

When a child subject is visited (/subject/forgiving-others), the template renders the same way — only the input slug changes. The breadcrumb chain extends to show the parent.

Personalization in the dynamic content slot

The dynamic content slot is where the Library does its work on Subject pages. Signals that influence what shows:

  • inferredFaithStage — Seekers see different teachings than Mature Believers, even on the same subject
  • inferredLifeStage + inferredLifeEvents — A user with high family-conflict signal sees forgiveness content framed for family contexts
  • visitedMinistries — Boosts content from ministries the user already trusts
  • watchPatterns.formatAffinity — Sermon-leaners see sermons first; devotional-leaners see devotionals first
  • preferredSessionMin — Tiebreaker on duration when relevance is close
  • Already-watched suppression — completed content sinks lower or hides entirely

Cold-start behavior

First-visit users get the same template but with the personalized rail collapsed or absent. The Library returns a single ContentGrid sorted by popularity. Sub-subject chips are shown so users can self-narrow. As signals accumulate, the personalized rail starts to appear above the grid.

Empty & edge states

Subject has < 6 published items

Hero shows. Content slot renders single ContentGrid with all available items. No personalized rail (not enough pool to rank from). MinistryRail may be empty — hidden if so.

Subject has no scripture anchor

ScriptureBlock slot renders empty and is collapsed. Pastoral Board notified to backfill.

Subject has no editorial framing

TextBlock slot collapses. Page still renders normally. Editorial team flagged.

Subject not found

404 with a fallback page showing top subjects and a search prompt.

Sensitive subject (suicide, abuse, addiction)

Page renders with a crisis-resources CTABlock at the very top. Sub-subject filter pre-set to safe defaults. Editorial framing is mandatory for these subjects (validation enforced at publish).
